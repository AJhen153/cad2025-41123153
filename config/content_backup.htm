openduck

先將每個會動的地方（類似人類的關節、膝蓋）設置座標系及軸。

左鍵工具，下拉找到Tool，再點擊Export as URDF。

我們以機器人的軀幹為整個機器人的主座標（robot_base），設定完主座標及組成軀幹的零件後，增加三個分支，分別為機器人的Neck、L_Hip、R_Hip。

點選分支進行座標、軸的命名、選取旋轉方式及設定並選取其組成的零件。

這是我們設定完的樹狀圖，完成設定後點選Preview and Export。

確認輸出的資料是否與座標、軸的設置相同，若無誤就點選Next進行下一步。

再次確認輸出資訊，無誤點選Export URDF and Meshes，建議新增資料夾儲存。

搜尋cmd打開命令提示字元，打上指令python -m pip install urdf2webots讓電腦安裝轉檔資源。

再打以下指令讓URDF檔轉成proto檔，路徑的部分需要自行更改。指令：python C:\Users\ASUS\AppData\Local\Programs\Python\Python314\Lib\site-packages\urdf2webots\importer.py --input D:\DUCK-1126\Duck\Duck\urdf\Duck.urdf --output D:\DUCK-1126\Duck\Duck\urdf\Duck.proto

成功的話就會在目的地資料夾看到生成一個proto檔。

打開webots並新開一個專案。

把剛剛生成的proto檔丟到新建的專案資料夾裡的protos資料夾裡並Reload World。

Add New新增一個Robot。

在Robots下找到children並點擊Add New。

在RPOTO nodes(Current Project)下可以看到我們的機器人，點選並點擊Add。

我們的機器人成功匯入，但有錯誤警告，目前正在排除中。

我們在專案資寮夾裡面的controllers資料夾裡新增名為duck_control的資料夾，並在裡面新增名為duck_control的py檔，用於寫入控制機器人的程式做使用。

Reload World後，我們在新增的機器人分支下找到controller"generic"點擊Select，可以找到剛剛新增的duck_control，點選並點擊OK後，右邊就能進行程式編輯。

根據AI的回答，Webots跳出的紅字警告與三角網格和節點過多有關，解決辦法有兩個，第一個是減少匯入的模型，第二個是將所有模型的三角網格都減少，為保萬全，我們兩個兩個都有做。

首先打開Blender將要減少網格數的模型STL匯入。

匯入後可以在右下角看到目前這個模型的網格總數以及網格類型。

點擊右邊藍色板手的符號，再點擊新增修改器。

游標移動到生成，再點擊大量毀滅。

在比率那一欄做更動，讓下方的網格數減少到一定程度。

完成後匯出STL。

其他的模型依照上述步驟減少三角網格，完成後到存放STL的資料夾將修改過的檔案取代原本的檔案。

打開Webots並Reload World後可以看到和網格、節點相關的警告消失了，其他的紅字警告正在排除中。
